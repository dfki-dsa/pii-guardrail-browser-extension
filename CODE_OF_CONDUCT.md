# Code Of Conduct

Privacy Guardrail contributors and maintainers are expected to keep project spaces focused, respectful, and safe for technical collaboration.

## Expected Behavior

- Use welcoming and inclusive language.
- Be respectful of differing viewpoints and experience levels.
- Keep criticism specific to ideas, code, documentation, and process.
- Accept responsibility for mistakes and correct them when needed.
- Protect user privacy when discussing examples, reports, screenshots, and logs.

## Unacceptable Behavior

- Harassment, intimidation, or discriminatory language.
- Personal attacks or sustained disruption of project discussions.
- Publishing private information without explicit permission.
- Sharing real personal data, secrets, private prompts, private responses, or private documents in public project spaces.
- Pressuring maintainers to accept privacy-invasive features that conflict with project policy.

## Reporting

For behavior concerns in public project spaces, contact the maintainers through the support channels in `SUPPORT.md`. For sensitive security or privacy concerns, use the private reporting process in `SECURITY.md`.

Maintainers may remove comments, close issues, reject contributions, or restrict participation when behavior conflicts with this code of conduct.
